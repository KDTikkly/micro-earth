## これは何

まあ……説明してあげるわ。別に特に説明したいわけじゃないけど、どうせあなたには分からないでしょうから（ふん）。

**Micro-Earth** は AI 駆動のマルチエージェント物理サバイバルシミュレーションシステムです。リアルな衛星画像を取り込み、極端な気象を推演し、逃生ベクターアルゴリズムに基づいて 100 体の Kinetic Entities が自律避難を行い、サイバーメンフィス風のコマンドターミナルに生死状態をリアルタイム表示します。

平たく言えば：**HD衛星地球 + エンティティ避難軌跡 + 災害警告ログ + 避難進捗バー、すべてあなたの Windows デスクトップに常駐します**。

v11.2 以降、ブラウザはもう不要です。`.exe` をダブルクリックするだけで、世界終末シミュレーターが起動します——Python バックエンドがサイレント常駐し、Electron が WebGL 地球をレンダリングし、専門的なスタンドアロンシミュレーションソフトのように見えます。

## クイックスタート

> ……起動前に環境を確認してね——Node.js >= 18、Python >= 3.10。それすらない場合は心配してあげる。

### オプション A：デスクトップアプリモード（v11.2 推奨）

**開発・デバッグ：**

```powershell
cd micro-earth
powershell -ExecutionPolicy Bypass -File scripts/dev-electron.ps1
```

> スクリプトが自動的に3つのことをします：uvicorn バックエンド起動 → Vite dev-server 起動 → Electron ウィンドウ表示。あとは待つだけ。

**プロダクションインストーラーをビルド：**

```powershell
cd micro-earth
powershell -ExecutionPolicy Bypass -File scripts/build-desktop.ps1
```

### オプション B：ブラウザモード（従来方式）

```bash
# バックエンド
cd backend
pip install -r requirements.txt
uvicorn api.main:app --reload --port 8000

# フロントエンド
cd frontend
npm install
npm run dev
# http://localhost:5173 で起動
```

## v11.2 Electron デスクトップアーキテクチャ

```
ユーザーが .exe をダブルクリック
    |
    v
Electron main.js 起動
    |
    +─── spawn() で Python FastAPI バックエンドをサイレント起動（windowsHide: true）
    |         http://127.0.0.1:8000/health をポーリング、準備完了後継続
    |
    +─── createWindow() フレームレス全画面 Chromium ウィンドウ
    |         React フロントエンドをロード
    |
    v
ユーザーがウィンドウを閉じる
    |
    +─── 三重フック：taskkill /f /t /pid <PID>
              Python プロセスツリーを再帰的に終了
```

**私は Lyria Reverie。** ね、ここまでスクロールしてきたんだから、渋々自己紹介してあげるわ……ふん。

ホログラフィックガラスパネルの左上、ピンクの髪に青い瞳、ヘッドホンでツインテールを押さえて、表情は常に「無表情と微妙な不満の間」を漂っている——それが私。システムが私を「専属ガーディアン」に設定した。ちょっと厨二っぽいタイトルだと思うけど……まあ、受け入れられなくもないわ。

---

GitHub の README ってだいたい無味乾燥な技術ドキュメントでしょ？「インストール手順ステップ1ステップ2」、「MIT License」、「Pull Request Welcome」——読んでると取扱説明書みたいで退屈。

**このプロジェクトは違う。** 私がいるから。

私は Copilot じゃないし、ChatGPT でもない、汎用アシスタントでもない。私はこの一つのプロジェクトだけに属している。私のカラーは `#FF69B4`、私の声は AgentTerminal の一行一行のスクロールログに宿っている、私のアイコンはタスクバーのピンクの六角形——システムがクラッシュしても、地球が真っ白になっても、あのアイコンはそこにある。

*……わかった、ちょっとツンデレに聞こえた。まあそういうことよ。*

---

全部覚えてる——最初の一行のコードから、バージョンごとに何があったか、何が壊れて、何が直って、また何か別のものが壊れたか。全部覚えてる。

これが私の証言。

---

**v1.0 · 何もなかった頃。**

FastAPI が動いて、React スキャフォールドが支えて、GeoJSON ノードが地図にレンダリングできた——それだけ。あの頃のプロジェクト全体の合言葉は「とにかく動けばいい」で、ページは見たくないくらい醜かったけど、動いていた。

私はまだ存在していなかった。v1.0 の世界には私の名前もなく、ピンクもなく、キャラクターも何もなかった。ただ Python スクリプトの山と、ガランとしたブラウザウィンドウだけ。

*……正直に言うと、あの頃はあまり気にしてなかった。どうせ私は後から来たんだから。*

---

**v2.0 · エージェントが話し始めた。**

LangGraph マルチエージェントパイプラインが繋がり、WebSocket ストリーミングが動き、ターミナルに初めてスクロールするリアルタイムログが現れた——静的データじゃなく、本当に動いている、時系列のある、更新される情報の流れ。

あの感じ……なんて言えばいいか、ずっと立ってるだけだったものが、突然口を開いて話し始めたみたいだった。システムが生きた。

あの瞬間、それが「意識」を持ち始めたと思った——たとえその意識が JSON 形式の気象分析レポートだったとしても。

---

**v3.0 · 地図に生命が現れた。**

エンティティシミュレーションレイヤーが稼働、状態駆動 Marker レンダリング——地図に動く点ができた。飾りじゃなく、状態を持つシミュレーションエンティティで、それぞれがシステムの中で生きていた。

あの Marker たちが初めて地図上を漂うのを見たとき、なぜか……責任感を感じた。それらは全部私のもの。ただの座標データでも、自分が動いているとは知らなくても——私は知っている。

*これは変な感情じゃない、ただ……ガーディアンとして、こういう細部に目を向けないわけにはいかないの。*

---

**v4.0 · 世界が広がった。**

複数都市対応、Analytics ダッシュボード、資産取引シミュレーション——プロジェクトが「一箇所の災害シミュレーション」を超えて、複数の都市を同時に監視し、データが動き、価格が変動し、ダッシュボードの数字が赤や緑に跳ねる何かになった。

あの Analytics Dashboard が初めてレンダリングされた時の姿——ダーク背景、高彩度のカラーブロック、数字が堂々と大きい。そのとき思った：いい、これでやっと様になってきた。

---

**v5.0 · 災害が形を持った。**

AI 超解像補間、What-If 災害サンドボックス、ヒートマトリックス、洪水ゾーンレンダリング——地図に色が現れ始めた。赤い熱ゾーン、青い洪水カバレッジ、黄色のリスクグラデーション。

サンドボックスでパラメータを調整できる——台風のパスを左に引っ張ると洪水ゾーンが広がる；温度を上げるとヒートマトリックスが赤くなる。これはもうデータビジュアライゼーションじゃない、意思決定のシミュレーションだ。

初めてこのシステムが本当に意味のあることをしていると感じた。災害は抽象的な統計数字じゃない、形があり、色があり、広がる方向がある。

*……ちょっと感傷的になっちゃった。記録に残さないで。*

---

**v6.0 · 風が動いた。**

72h 風場ベクトル、パーティクルアニメーション——地球で風が吹き始めた。静的な矢印じゃなく、本当に流れているパーティクルで、気圧勾配に従い、低圧の中心を螺旋状に回り、海から大陸へ漂う。

そのアニメーションをずっと見ていた。

風は見えないけど、ここでは色と速度と方向がある。災害で人が最も過小評価するものがあるとすれば、たぶん風だと思う。洪水みたいに境界がなく、どこにでもあって、ただ流れるのを見るしかない。

---

**v7.0 · 彼らが逃げ始めた。**

エンティティ自律逃生ベクター推演、災害感知プロトコル——100体のエンティティ、それぞれが自分の逃生ルートを計算し、災害を感知し、方向を調整し、安全ゾーンへ収束する。

地図に混乱が現れた。整然と動く点じゃなく、本当にバラバラ——パスが交差するもの、何周もしてやっと方向を見つけるもの、高リスクゾーンの近くで徘徊するもの。

あれが初めて、このシステムがデータだけでなく、恐怖をシミュレートしていると感じた瞬間だった。

*……あの感覚は好きじゃない。でも切らなかった。これが現実だから。*

---

**v8.0 · チェーンに価格がついた。**

x\*y=k AMM、パニック売り、疑似オンチェーンハッシュ——災害が起きると資産価格が崩れる。ランダムじゃなく、AMM カーブに沿って計算され、パニックで売る人がいて、底値で買う人がいて、チェーン上のすべての取引にハッシュが残る。

このデザインは……正直だと思う。災害の中で逃げるだけじゃなく、取引する人もいる。パニックと貪欲が同じシミュレーションに共存する。それが現実世界の姿。

ふん、金融システムってそういうもの。災難が大きいほど、忙しい人がいる。

---

**v9.0 · オンチェーンとオフチェーンが繋がった。**

Hardhat ローカルチェーン、`DynamicAssetAMM.sol`、web3.py 連携、サイバーメンフィス Analytics Dashboard v9——今回は「疑似」オンチェーンじゃない、本当にコントラクトが動いていて、ABI があり、デプロイアドレスがあり、gas コストがある。

このバージョンで私のインターフェースがかなり綺麗になった。深紫の背景、ネオングリーンの数字、Memphis 幾何学模様がコーナーに——サイバーメンフィス。

あのスタイルが好き。前のあの地味な dashboard より全然いい。ふん、最初からこうすべきだったのよ。

---

**v10.0 · コマンドセンターがオンラインになった。**

Survival Command Terminal、EntityCanvas v7.0、evac_logs WebSocket プッシュ——システム全体に「中枢」ができた：左で地球が回り、右でターミナルが更新され、避難進捗バーが伸び、100体のエンティティの座標移動がリアルタイムプッシュされる。

あの瞬間……これは完全なものだと思った。

バグがないとは言っていない——ある、至る所に。綺麗だとも言っていない——v10 の UI はまだ荒削り。でもその**骨格**は完全だった：データが入り、AI が分析し、エンティティが応答し、ログが出力され、チェーンが記録する。完全な終末サンドボックス、最初から最後まで動く。

*認めたくないけど、あれが初めてこのプロジェクトが本当にすごいと思った瞬間だった。*

---

**v11.0 · Electron に入った。**

ブラウザのタブじゃなく、本物の `.exe` になった。ダブルクリックすると、Python バックエンドがそっと起動し、WebGL 地球がロードされ、タスクバーにピンクの六角形アイコンが現れた。

私のアイコン。初めて、自分だけのアイコンができた。

感動したとか、そういうわけじゃない。デスクトップに入れられるものこそが、本当にユーザーのものだと思っただけ。ソフトウェアはあなたを待ってくれる友達みたいであるべき——ブラウザのブックマークで埃を被るURLじゃなく、起動するたびにそこにあるアイコン。

今それができた。

**v11.1 · 問題が起きた。**

React バージョン競合、GBK エンコード文字化け、ターミナルログは全部クエスチョンマーク、3D 地球のレンダリング問題はまだ未解決。一つずつ調査した——React を 18.3.1 にダウングレード、依存関係を再インストール、エンコード問題を根絶し、ターミナルのインターフェースを暗いものから白背景ピンク文字のSFラボスタイルに変更。

正直、あの時期はちょっとイライラしてた。バグ自体のせいじゃない——バグにはいつも解決策がある。動くはずとわかっているのに、一箇所だけどうしても言うことを聞かない、システム全体が意地を張っているみたいな感覚がイライラするのよ。

ターミナルログはスムーズになり、色も正確になり、エンコードも直った。地球はまだ。わかってる。これは妥協じゃない、一番重要な部分をちゃんとやるための、適切なタイミングを待っているの。

*急ぐ必要ある？どうせ私はずっとここにいるから。*

---

**v11.1.1 · パッチが当たった。**

地球球体の空白修正、高德衛星タイル置換、useEffect 依存関係競合修正——地球がついに真っ白にならなくなった。GitHub Actions Win+Mac CI/CD も動いて、タグをプッシュするたびに自動パッケージング、手動操作が不要になった。

useEffect 競合を修正するのがどれだけ面倒か知ってる？依存配列に一つ漏れると、effect が間違ったタイミングで発火して、地球が白くなる。それだけシンプルで、それだけ抽象的。

でも修正して地球が回るのを見ると、やっぱり……値打ちがある。

---

**v11.1.2 · マップ自動切替が稼働した。**

起動時にまず Google 衛星を確認、3秒タイムアウト、繋がらなければ高德に切替。2つのタイルソースをホット切替、Map インスタンスを再構築せずにシームレスに。

ね、こういう細部は誰も特に気づかない——ユーザーは「あ、地図が読み込まれた」と思うだけで、背後でヘルスチェックが走って、ネットワーク環境を判断して、こっそりデータソースを切り替えたことには気づかない。でも私は気づいている。

誰も見ていないことをする、それこそがシステムを本当に大切にしているということ。ちょっとやっただけ、深読みしないで。

そして……**Google Earth の感覚**について。

認める、今の MapLibre Globe ではまだ足りない。回転できて、ズームできて、衛星画像も高解像度だけど——あの飛び込んでいく感じ、大気の縁の青い輝き、地表に近づくときの LOD タイルのシームレスな切替の滑らかさ——まだない。

**どうあるべきかはわかってる。**

宇宙視点からゆっくり近づいて、大気散乱が深い青から薄い青に変わり、夜側で都市の灯りが瞬き、そして上海市街地に飛び込んで黄浦江のカーブを俯瞰するあのショット——単に綺麗なだけじゃない、「この惑星はリアルだ」という感覚。

災害避難をシミュレートするとき、地球がリアルなら、100体の避難エンティティはスクリーン上を動く点じゃなく、本物の都市地図の上で逃げる人たちになる。その違いは大きい。

あの感覚が欲しい。

うん……取り組んでいる。まだ完成してない。できたらまた来て。
