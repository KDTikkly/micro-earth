## ◈ SECTION 0 · 系统概述

Micro-Earth 是一套**受真实物理环境驱动的多智能体气象推演数字孪生沙盘**。

呐，说简单点——它会用真实的气象数据，驱动一群 AI 智能体在虚拟城市里模拟极端天气事件的影响。台风来了怎么样？气温暴涨会怎样？这里全部可以测试！

```
  [操作员] ──▶ [3D 地球 UI] ──▶ [WebSocket] ──▶ [LangGraph 智能体]
                                                         │
                   ◀── 风场 / 热力图 / 实体状态 ──────────┘
                                                         │
                                         [Open-Meteo 气象 API]
```

**核心技术栈：**

| 层级 | 技术 | 作用 |
|------|------|------|
| 3D 渲染引擎 | `react-globe.gl` + Three.js | 地球可视化 |
| 智能体框架 | LangGraph (Python) | 多节点气象推演 |
| 状态管理 | Zustand | 前端全局数据流 |
| 实时通信 | WebSocket `/ws/agent-stream` | 后端推流 |
| 气象数据 | Open-Meteo API | 实时天气场 |
| 前端框架 | React 19 + Vite + Framer Motion | UI 渲染 |

## ◈ SECTION 1 · UI 模块解析

### 1-A · 左侧控制列

#### ◉ Lyria A.I. 立绘面板

呐，左上角那个全息玻璃框里的，就是我啦～ `CLASSIFIED · LYRIA A.I.` 的标签说明系统正在安全授权模式下运行，才不是随便什么人都能看到的哦。

#### ◉ 系统状态矩阵（SYSTEM STATUS MATRIX）

6 个孟菲斯色块徽章，一眼就能看出哪个子系统挂了——这个设计是我最喜欢的部分之一，才不是因为上面有我的名字！

- `FRONTEND :3000` — 前端服务端口
- `MAP GLOBE·3D` — 3D 地球引擎状态
- `OPEN-METEO WEATHER·API` — 气象数据源
- `NODES Xpts` — 当前推演网格节点数量

#### ◉ What-If 假设推演沙盘

哼，这个才是整个系统最厉害的部分！你可以亲手"篡改"天气参数然后看世界会怎样，是不是有点坏坏的感觉～

- **温度偏移滑块（TEMP OFFSET）**：范围 `-5°C ~ +5°C`
  - 向右拖 → 模拟全球变暖
  - 向左拖 → 模拟极寒异常
- **降水倍率滑块（PRECIP MULTIPLIER）**：范围 `0.5x ~ 2.0x`
  - 超过 1.5x → 触发洪涝风险推演
  - 低于 0.7x → 触发干旱风险评估
- **`APPLY INTERVENTION` 按钮**：把参数注入下一次请求
- **`RESET` 按钮**：归零所有参数

> ⚠ 呐，改完参数一定要先点 `APPLY INTERVENTION`，再点 `▶ FETCH` 哦！跳过这步的话……才不会帮你解释为什么没变化呢。

#### ◉ 气象数据面板（WEATHER / RISK 卡片）

实时显示当前城市的温度和降水概率。`RISK INDEX` 红色闪烁的时候，说明这个区域已经进入极端气象风险区——这时候终端里的智能体也会变得很忙哦。

### 1-B · 中央地图区（3D GLOBE ENGINE）

呐，这颗会转的地球是整个系统的核心！我超喜欢看着它转的……才不是什么特别的原因！

#### ◉ 3D 地球本体

- **蓝色大理石贴图**：静态全球贴图（Phase 9 规划升级为动态瓦片！）
- **彩色高度柱**：气象节点温度热点，颜色越红代表越热：
  - `#00CFFF` 冰蓝 → ≤5°C
  - `#AAFF00` 荧光绿 → 15~22°C
  - `#FF6600` 霓虹橙 → 28~33°C
  - `#FF0055` 霓虹粉 → >33°C 极热！
- **彩色弧线**：洪涝风险连接线（紫→粉），从当前城市指向高危区域
- **右键 / 触控拖拽**：旋转地球视角
- **滚轮**：缩放（已设置最大缩放限制，保护贴图不被放成马赛克）

#### ◉ AUTO-ROTATE Toggle（右下角黄色按钮）

这个是我专属设计的按钮哦，才不是随便加上去的～

- 点击切换地球自动旋转 `ON / OFF`
- `ON` → 明黄底黑字，地球慢慢转圈圈
- `OFF` → 暗色底，手动旋转到你想看的任何角度

#### ◉ FETCH 按钮（右上角绿色）

点这个按钮，才能真正"唤醒"系统开始推演。
状态流转：`IDLE → CONNECTING → RUNNING → IDLE`

#### ◉ 覆盖徽章

| 徽章位置 | 内容 |
|----------|------|
| 左上 | 风速状态（当前时间帧平均风速） |
| 右上 | 洪涝区域数量 / 无风险提示 |
| 左下 | 数据节点数量 / AWAITING DATA |
| 底部 | 实体统计（NORMAL / STRESSED / PANIC） |

### 1-C · 时空播放器（TEMPORAL SCRUBBER）

> 位于 3D 地图正下方，紫色边框区域

呐，这个时间轴是我最喜欢演示给别人看的功能！可以把未来 72 小时的风场变化一帧帧播放出来——有没有一点"预知未来"的感觉？

- **▶ PLAY 按钮**：自动逐帧播放（300ms/帧）
- **⏸ PAUSE 按钮**：暂停在当前帧
- **滑块**：手动跳转到任意小时（+0H 到 +71H）
- **右侧 +XH 标签**：当前帧时间偏移
- **AVG / MAX 风速**：当前帧的风速数据

**风速图例（左下）：**

| 颜色 | 等级 | 速度 |
|------|------|------|
| `#9370DB` 电子紫 | CALM | < 3 m/s |
| `#00BFFF` 电光蓝 | BREEZE | 3–8 m/s |
| `#FFEE00` 明黄 | STRONG | 8–15 m/s |
| `#FF6600` 霓虹橙 | GALE | 15–25 m/s |
| `#FF0055` 霓虹粉 | TYPHOON | > 25 m/s |

## ◈ SECTION 2 · 标准操作流程（SOP）

呐，照着这个顺序来，保证不会出错！

```
STEP 1  ▶  顶部导航栏 [RGN] 下拉框选择目标城市
           可选：深圳 / 北京 / 上海 / 成都 / 东京 / 纽约

STEP 2  ▶  （可选）左侧 What-If 沙盘调整气象干预参数
           一定要点 [APPLY INTERVENTION] 保存！

STEP 3  ▶  点击地图右上角绿色 [▶ FETCH] 按钮
           等待状态变为 RUNNING

STEP 4  ▶  观察右侧终端滚动日志：
           · "event: geojson"    → 气象网格数据已加载
           · "event: entities"  → 智能体实体初始化完毕
           · "event: heatmap"   → 热力矩阵 + 洪涝区域就绪
           · "event: windfield" → 72H 风场矢量场就绪
           · "event: done"      → 推演完成！

STEP 5  ▶  地球上观察：
           · 彩色高度柱 = 温度热点分布
           · 彩色弧线   = 洪涝风险传播路径
           · 粒子流动   = 实时风场方向和强度

STEP 6  ▶  拖动时空播放器，回溯 / 预演 72 小时风场变化

STEP 7  ▶  点击 [AUTO-ROTATE: ON/OFF] 从不同角度查看数据
```

## ◈ SECTION 3 · 异常处理

哼，系统出问题不要慌，让我来帮你——才不是因为担心你嘛！

### ◈ 异常 A：终端红色 `[ERROR] WebSocket 连接失败`

**症状**：右侧终端出现红色报错，状态徽章变红

**原因**：后端 FastAPI 服务没跑起来，或 8000 端口被占了

**处置方法**：
```bash
# 启动后端
cd micro-earth/backend
uvicorn main:app --reload --port 8000

# 确认启动后刷新前端，重新点 [▶ FETCH]
```

---

### ◈ 异常 B：地图显示 `◈ MAP ENGINE ERROR`

**症状**：地图黑屏，中央出现红色错误提示

**原因**：WebGL 初始化失败（浏览器不支持或 GPU 内存不足）

**处置方法**：
1. 点黄色 `↺ RETRY` 按钮重新初始化
2. 确认浏览器支持 WebGL（可访问 `webglreport.com`）
3. 关闭其他占用 GPU 的标签页后刷新

---

### ◈ 异常 C：地球加载出来是纯黑球

**症状**：Globe 渲染了但没有贴图，纯黑

**原因**：`unpkg.com` CDN 贴图加载失败（网络问题）

**处置方法**：
1. 检查网络（需能访问 `unpkg.com`）
2. 可将 `EarthMap.jsx` 的 `globeImageUrl` 替换为本地贴图路径

---

### ◈ 异常 D：What-If 改了参数但没有变化

**症状**：调了滑块但地球数据没更新

**原因**：漏掉操作步骤了！

**处置方法**：
```
修改滑块 → APPLY INTERVENTION → ▶ FETCH（重新建立 WebSocket）
```

呐，说了要按顺序来的嘛……

---

### ◈ 异常 E：时空播放器 PLAY 按钮是灰的

**症状**：`▶ PLAY` 不可点（变暗了）

**原因**：还没有获取到 72H 风场数据

**处置方法**：
1. 先点 `▶ FETCH` 获取完整数据集
2. 等终端出现 `event: windfield` 日志
3. 播放器自动解锁
